# 高阶架构设计

## 1. 架构愿景

构建AgileTDFlowDev系统，提供项目级别的AI辅助开发管理能力，支持从需求输入到迭代执行的完整工作流。

**关键能力：**
- 项目级规划工作流（整合现有skills）
- 中文输出文档自动生成
- 项目隔离的文档组织结构
- 团队风格文档模板系统
- 代码库上下文智能理解

**非功能需求：**
- 易用性：降低AI编程使用门槛
- 可扩展性：支持未来新增skill接入
- 一致性：所有输出符合团队文档规范
- 智能性：AI能基于上下文给出实际可行的方案

## 3. 核心组件

### 3.1 项目规划子系统 (Project Planning Subsystem)
**职责：** 整合project-planning skill，管理项目级规划工作流
**接口：**
```python
class ProjectPlanner:
    def create_project_plan(self, req_file: Path) -> ProjectPlan
    def update_iteration_status(self, iter_num: int, status: IterStatus)
    def get_epic_progress(self, epic_id: str) -> Progress
```

### 3.2 迭代执行子系统 (Iteration Execution Subsystem)
**职责：** 管理迭代生命周期，协调brainstorming/writing-plans/execution
**接口：**
```python
class IterationManager:
    def start_iteration(self, iter_num: int, epics: List[Epic]) -> IterContext
    def generate_iter_spec(self, context: IterContext) -> IterSpec
    def complete_iteration(self, iter_num: int, retrospective: Retro)
```

### 3.3 文档管理子系统 (Document Management Subsystem)
**职责：** 按项目组织文档，管理中文模板，控制版本
**接口：**
```python
class DocumentManager:
    def init_project_dir(self, project_name: str) -> ProjectPath
    def render_template(self, template: str, context: dict, lang: str = "zh") -> Document
    def link_documents(self, source: DocRef, target: DocRef)
```

### 3.4 代码库理解子系统 (Codebase Understanding Subsystem)
**职责：** 分析代码库结构、依赖、约定，为AI提供上下文
**接口：**
```python
class CodebaseAnalyzer:
    def investigate_structure(self, root: Path) -> CodebaseMap
    def extract_conventions(self, files: List[Path]) -> ConventionSet
    def suggest_integration(self, feature: FeatureDesc) -> IntegrationPlan
```

## 4. 数据流

**项目启动流程：**
1. 用户输入项目需求文档
2. ProjectPlanner分析需求复杂度
3. 如需架构设计，调用架构设计模块
4. 生成项目计划文档（中文）
5. 初始化项目目录结构
6. 存储文档并建立索引

**迭代执行流程：**
1. 用户启动迭代N
2. IterationManager加载当前Epic
3. CodebaseAnalyzer分析相关代码上下文
4. 生成迭代规格文档（iter-spec，中文）
5. 执行brainstorming → writing-plans → execution
6. 迭代完成后更新项目计划状态

## 5. 技术选型

| 组件 | 选择 | 理由 |
|:---|:---|:---|
| Skill框架 | Claude Code Superpowers | 复用现有基础设施 |
| 文档格式 | Markdown + YAML Frontmatter | 团队现有规范 |
| 模板引擎 | Jinja2 (Python) | 灵活、成熟 |
| 项目目录 | 文件系统层次结构 | 简单、可版本控制 |
| 代码分析 | Tree-sitter + AST | 多语言支持、精确 |

## 6. 模块依赖

| 模块 | 依赖 |
|:---|:---|
| ProjectPlanner | DocumentManager |
| IterationManager | ProjectPlanner, CodebaseAnalyzer, DocumentManager |
| DocumentManager | 无（基础模块）|
| CodebaseAnalyzer | 无（基础模块）|
| AgileTDFlowDev Main | 所有子系统 |
